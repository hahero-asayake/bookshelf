# T10: 公開モードのデザイン

状態: 未着手 / 依存: T09 / **モック承認ゲートあり (ユーザ確認なしで実装に入らない)**

## 目的

T09 で機能する公開モードに、閲覧者向けのデザインを与える。方向性 (旧検討メモより): **ラウンジ (落ち着いた閲覧体験) + 雑誌キュレーション (見せたい本棚を演出)** の組み合わせ。読者ペルソナ: 「美しく蔵書を眺められる、読書ブログ感」(01_要件定義書)。

## 進め方 — モックファースト (必須)

1. `mockups/public-mode.html` を新規作成し、**自己完結 HTML で 2〜3 案** (アプリのデザイントークンを再現):
   - 共通要素: ヒーローヘッダー (サイト名 = hahero の本棚であること + 一言紹介) / 公開本棚のナビ / 本グリッド / 本詳細 (読み取り専用: 表紙・タイトル・著者・星・短文メモ・長文メモ・Amazon リンク)
   - 案の振り幅の例: A) 現編集 UI の 3 ペインを閲覧用に簡素化 (慣れた構造) / B) 1 カラム雑誌風 (本棚ごとのセクション + 大きめ表紙 + キュレーション文) / C) A+B ハイブリッド (上部に特集、下に棚ブラウズ)
   - PC とモバイル両方のスクリーンショットを撮る
2. **ユーザに提示して 1 案 + 修正点を確定** (AskUserQuestion)。承認前に index.html / css を触らない
3. 承認後に実装

## 実装の指針 (承認案に従う。以下は共通要件)

- 同一コードベース。`body.public-mode` スコープの CSS + 公開専用の描画分岐で実現し、**編集モードの DOM/CSS への影響ゼロ**を保つ
- 読み取り専用の本詳細: 星は表示のみ (クリック無効)、メモは表示のみ、所属本棚 chip はナビとして機能
- 公開者情報: 表示名「hahero」(**本名を出さない** — vault 運用ルール)。紹介文は `public/settings.json` 由来のフィールドを新設するか定数 (実装時にユーザへ確認)
- 長文メモ: frontmatter を表示しない (T04 のヘルパを流用)
- フッターにアフィリエイト開示 (既存「このアプリについて」の文面を流用) と「Powered by bookshelf」リンク
- パフォーマンス: 公開ページも content-visibility 方針を踏襲

## 受け入れ基準

1. ユーザがモック 1 案を承認済みで、実装がモックと一致 (PC 1280px / モバイル 390px のスクリーンショット比較)
2. 公開モードで編集系 UI が出ない (T09 基準の再確認)
3. 編集モードの見た目・挙動が無変更 (T05 smoke green + 目視)
4. 本名が公開ページのどこにも出ない (grep + 目視)
5. console エラー 0
