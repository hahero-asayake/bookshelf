# P1: 公開システム再設計 — 静的 SSG ＋ 公開ページ単位 (MVP)

状態: 🟡 設計合意済み・未着手 (2026-06-13) / 依存: 既存の公開選択・個人情報除去・GitHub push 経路 (流用) / 置換: T09/T10
方針: ADR-030 / 別紙 `09_公開システム設計` (vault)。**ユーザ合意済みの確定事項に従う。コア拡張はプラグインに逃がし、コアは単純に保つ。**

## 目的

「アプリに蓄積した情報を手軽に公開」を、**静的ページ生成 (SSG)** で実現する。
旧 `?u=` (公開 repo のデータをアプリ丸ごと起動して描画) は**廃止**し、アプリ内で**自己完結の静的 HTML/CSS** を生成して公開先 (当面 GitHub) へ置く。管理単位は **「公開ページ」**。

## 確定事項 (動かさない)

- **ページ＝1スタイル**が基本モデル。スタイルが必要な対象・項目を宣言し、ユーザが埋める。
- **「自由度」= スタイル/プラグインで拡張**。ブロックビルダー・サイト化・独自ドメインは**コアに作らない** (必要なら“自由構成スタイル”や別プラグインで)。
- **公開はオプトイン**: ページに載せた対象・ON にした項目だけ出力 (旧オプトインの思想を継承)。
- **公開先は当面 GitHub のみ**。ただし `PublishTargetAdapter` で抽象化し、将来 Cloudflare(BYO/運営) を差し込めるようにする。
- レビューは**機能優先** (対象・項目が正しく反映 / 静的に成立 / 個人情報非漏洩)。デザイン磨きは後。

## サブタスク (上から順に)

### P1-1 公開ページのデータモデル & ストア
- `private/publish/pages.json` を新設。1 ページ = 
  `{ id, slug, title, intro, styleId, styleParams, select:{ shelves:[internalId], books:[asin], fields:{rating,memo,detailMemo,cover,author,amazon} }, lastBuiltAt }`
- `PublishPageStore` (CRUD: 一覧/作成/更新/削除/複製) を storage 経由で実装。private 配下なので公開には出ない。
- 受け入れ: 作成→保存→再読込で往復一致。複数ページを保持できる。

### P1-2 スタイル エンジン
- `PublishStyle` インターフェース: `{ id, name, declare(): {requires, fields}, render(ctx): { files: [{path, content, kind}] } }`。
  - `declare()` … 必要な対象 (本棚 N / 本 1 等) と入力欄 (タイトル・紹介文等) を宣言 → UI が動的にフォーム生成。
  - `render(ctx)` … 選択データ + params から**自己完結の HTML/CSS (+画像参照)** を返す。アプリ/プラグイン実行は載せない。
- スタイルは**内蔵プラグイン**として実装できる形にする (P2 で正式に plugin-api へ。P1 では内部レジストリで可)。
- 受け入れ: ダミー選択で render → 単体 HTML として開け、外部依存なしで表示。

### P1-3 公開ページ管理 UI
- 設定または専用画面に「公開ページ」一覧 + 新規作成。
- 作成フロー: ①スタイル選択 → ②`declare()` に従い対象 (本棚/本) を選択 + 公開項目トグル → ③スタイル別パラメータ入力 → ④**プレビュー** → 保存。
- 既存の「この本棚を公開する(isPublic)」チェックは**公開ページの対象選択へ統合** (or 廃止)。
- 受け入れ: 1 ページを GUI だけで作成・編集・プレビューでき、console エラー 0。

### P1-4 静的ジェネレータ + プライバシーガード
- ページ定義 → スタイル `render` → 生成物 (HTML/CSS/必要画像参照) をまとめる。複数ページのトップ (index) は**最小限**(一覧リンク)。サイト化はしない。
- 生成物に対し `_detectPrivateLeak` 相当を実行 (affiliateId/obsidian*/各種 hide フラグ/本名 を出力に出さない)。OFF 項目・非選択本は含めない。
- 受け入れ: 生成物の grep で個人情報・非公開項目・非選択本が**一切含まれない**。スナップショットなので「再生成」アクションで最新化できる。

### P1-5 公開先アダプタ (GitHub) + 旧モード廃止
- `PublishTargetAdapter` を定義 (`publish(files), listExisting(), delete()` 等)。GitHub 実装 = 既存の第2 GitHubAdapter + 削除同期 + 公開先 repo 選択 (今朝の UI を流用)。
- 公開実行 = 生成物を公開 repo へ 1 コミット push、削除同期で非公開化伝播、公開 URL を toast + どこかに常設表示 (通知消失対策)。
- 旧 `?u=` 公開モード描画分岐 (`loadDataPublicMode` 等) と公開モード専用 UI を**撤去**。
- 受け入れ: GitHub の public repo に静的ページが置かれ、その URL を直接開いて (アプリ起動なしで) 閲覧できる。SEO 用に各ページが実 HTML。

### P1-6 標準スタイル 5 個 + 機能検証
- ①本棚セクション型 ②本単体じっくり型 ③一覧ミニマル型 ④雑誌キュレーション型 ⑤フリーフォーム/ブロック型。
- 各スタイルで「対象・項目が正しく反映 / 単体 HTML で成立 / 個人情報非漏洩 / モバイル幅で崩れない」を確認。
- 受け入れ: 5 スタイルで各 1 ページ生成し上記を満たす。最終的にデフォルト同梱に残す集合はユーザが検証後に決める (実装側は全部出す)。

## 受け入れ基準 (P1 全体)

1. GUI だけで「公開ページ」を作成→GitHub へ公開→公開 URL を**アプリ起動なし**で閲覧できる。
2. 公開物に**個人情報・非公開項目・非選択の本が一切含まれない** (grep + 目視)。
3. 5 種のスタイルが機能的に成立 (対象/項目反映・静的成立・モバイル可)。
4. 旧 `?u=` 公開モードが撤去され、編集モードの見た目・挙動は無変更 (smoke green)。
5. 公開先は `PublishTargetAdapter` 抽象越し (将来 Cloudflare を足せる構造)。console エラー 0。

## 後続フェーズ (P1 では作らない)

- **P2**: スタイルを正式な**プラグイン API** 化 (描画フック署名・必要項目スキーマを plugin-api へ)。内蔵5種を再実装。
- **P3**: `PublishTargetAdapter` に **Cloudflare BYO** (ユーザ自身の CF API トークン)。
