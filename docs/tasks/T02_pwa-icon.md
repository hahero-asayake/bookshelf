# T02: PWA アイコン刷新

状態: 未着手 / 依存: なし

## 目的

現行アイコン (System.Drawing 自動生成の accent 地 + library グリフ) を、デザインされたアイコンに置き換える。favicon も統一する。

**確定したデザイン方向 (2026-06-12 ユーザ決定)**:
> Kindle のアイコン風。Kindle は「本を読む子供」だが、このアプリは「**本棚から本を取る子供**」

= フラットでミニマルなシルエットの人物 + やわらかいグラデーション地、という Kindle アプリアイコンの文法を踏襲しつつ、モチーフを「本棚に手を伸ばして本を取る子供」に置き換える。

## 実装手順

### Step 1: 候補 4 案の生成と提示

1. `mockups/icon-candidates.html` を作成。**インライン SVG で 4 案**を描く。全案共通:
   - 背景: accent 系グラデーション (`#5b6cff` → `#4856e6`、上が明るい)。角丸正方形プレビュー
   - 人物: 白 1 色のフラットシルエット (ディテールを削ぎ、輪郭で語る。顔・指は描かない)
   - 本棚・本: 白 or 白の半透明。本 1 冊だけ差し色 (薄い白系) にしてもよい
   - **maskable セーフゾーン**: 主要素を中央 80% の円内に収める
2. 案のバリエーション (構図違い):
   - **A**: 横向き全身の子供が 3 段の本棚に手を伸ばす (本棚は右、子供は左)
   - **B**: 背伸び・つま先立ちで上段の本に手を伸ばす子供 (人物を大きく、棚は上部 1 段のみ)
   - **C**: 棚から本を半分引き出した瞬間のクローズアップ (棚 2 段 + 腕と本だけ。最も抽象的)
   - **D**: 取った本を両手で掲げる子供 (動きと喜びのある絵)
3. 各案を 512 / 192 / 48px とマスク (円形クロップ) プレビューで並べる
4. Playwright でスクリーンショット → ユーザに提示し、**AskUserQuestion で 1 案選んでもらう** (微調整の要望もここで受ける)

### Step 2: 選定案の量産

1. 確定 SVG を `icons/source.svg` として保存 (今後の再生成元)
2. PNG 書き出し: `mockups/icon-export.html` (一時ファイル、終わったら _trash へ) に、確定 SVG を**正確なピクセルサイズの要素**として並べ、Playwright の **element screenshot** で出力:
   - `icons/icon-192.png` (192×192)
   - `icons/icon-512.png` (512×512)
   - `icons/icon-maskable-512.png` (512×512、**全面ベタ背景 + 主要素を中央 80% に縮小配置**した maskable 専用構図)
   - `icons/apple-touch-icon-180.png` (180×180、透過なし・全面背景)
   - `icons/favicon-32.png` (32×32。48px で視認確認してから出力)
   - 既存ファイル名が異なる場合は**既存の名前に合わせて上書き**し、manifest / head の参照と齟齬を出さない (まず `manifest.webmanifest` と index.html `<head>` の現参照名を確認すること)
3. device pixel ratio は 1 固定で撮影 (サイズずれ防止)。出力後、各 PNG の実寸を確認

### Step 3: 組み込み

1. `index.html` `<head>`: favicon link を新 PNG に変更 (絵文字 favicon を使っていた場合は置換)。`<title>` の 📚 テキストは**そのまま残す**
2. `manifest.webmanifest`: アイコンパス・purpose (`any` / `maskable`) が正しいか確認
3. `sw.js`: キャッシュバージョン定数をバンプ (旧アイコンのキャッシュ残り防止)
4. 旧アイコン PNG は `_trash/` へ (上書きでなくファイル名が変わった場合)

## 受け入れ基準

1. ユーザが 4 案から選択済みで、その案が反映されている
2. `manifest.webmanifest` の全アイコン URL が 200 を返し、実寸が宣言と一致
3. ブラウザタブの favicon が新アイコンで表示される
4. maskable 版: 円形クロップしても主要素が欠けない (icon-candidates.html のマスクプレビューで確認)
5. console エラー 0 / SW 更新フロー (上部バー「更新」) が壊れていない

PWA 再インストールでのホーム画面アイコン確認は**ユーザの実機作業** (報告時に依頼する)。
